## Finbot — Telegram‑бот для учёта личных финансов

`finbot` — это простой Telegram‑бот, который помогает вести учёт доходов и расходов в **Google Sheets**.  
Вы пишете (или надиктовываете) сообщение в Telegram — бот распознаёт сумму, дату, категорию и сохраняет операцию в таблицу.

- **Интерфейс**: Telegram‑чат с ботом.
- **Хранилище данных**: Google Sheets (отдельный spreadsheet с листами “Журнал” и “Категории”).
- **Интеллект**: внешний LLM (OpenAI‑совместимый API) и Whisper для расшифровки голосовых сообщений.

---

## Основные возможности

- **Быстрый ввод операций**
  - Текстом: `продукты 3000 вчера`, `зарплата 120000 5 числа`.
  - Голосом: надиктовали то же самое — бот распознаёт речь и обрабатывает текст так же, как обычное сообщение.
- **Автоматический разбор операций**
  - Определение **суммы**, **даты**, **категории** с помощью LLM.
  - При неуверенности бот помечает операцию как `pending` и предлагает выбрать/уточнить категорию кнопками.
- **Хранение в Google Sheets**
  - Все операции попадают в лист “Журнал”.
  - Категории хранятся в листе “Категории” и могут быть автоматически засеяны шаблоном.
- **Редактирование записей**
  - Команда `/edit` — просмотр и изменение последних операций:
    - смена суммы;
    - изменение даты;
    - смена категории;
    - отмена операции.

## Архитектура (в общих чертах)

- `app/main.py` — точка входа:
  - загружает настройки из `.env`;
  - создаёт Telegram‑бота и `Dispatcher`;
  - инициализирует Google Sheets‑клиент и репозитории (`JournalRepo`, `CategoryRepo`);
  - инициализирует LLM‑клиент и Whisper‑транскрайбер;
  - запускает `dp.start_polling(bot)`.

- **Telegram‑слой**
  - `app/telegram/bot.py` — фабрики для бота и диспетчера.
  - `app/telegram/handlers.py` — обработчики:
    - `/start` — проверка, что бот жив;
    - `/edit` — диалог редактирования записей по FSM (`EditJournalStates`);
    - текстовые сообщения — разбор через LLM/регекс и запись в журнал;
    - голосовые сообщения — транскрибация через Whisper, дальше тот же поток, что и для текста;
    - callback‑кнопки для выбора категорий и редактирования.

- **Доменные модели и сервисы**
  - `app/models/operation.py` — модель операции (`Operation`).
  - `app/services/ingest_service.py` — сборка операций из текста, fallback‑разбор суммы регексом.
  - `app/services/gpt_parse_service.py` — вызов LLM с подробным системным промптом и списком категорий.
  - `app/services/transcribe_service.py` — транскрибация голосовых сообщений через Whisper‑совместимый API.

- **Интеграция с Google Sheets**
  - `app/sheets/client.py` — обёртка над Google Sheets API (`SheetsClient`).
  - `app/sheets/journal_repo.py` — работа с листом “Журнал”:
    - добавление записей;
    - поиск дубликатов по Telegram `message_id`;
    - выборка и обновление последних операций пользователя.
  - `app/sheets/category_repo.py` — работа с листом “Категории”:
    - инициализация шаблонными категориями;
    - поиск/чтение категорий по id/имени.
  - `app/data/category_templates.py` и `app/data/categories.py` — преднастроенные категории и их коды.

- **LLM и промпты**
  - `app/llm/client.py` — минимальный OpenAI‑совместимый клиент для `/chat/completions`.
  - `app/llm/prompts.py` — список категорий и части промптов.

## Требования

- Python 3.11+ (рекомендуется).
- Аккаунт Google с доступом к Google Sheets.
- Созданный OAuth‑клиент для доступа к Sheets (JSON‑файл учётных данных).
- Токен Telegram‑бота от `@BotFather`.
- Доступ к LLM/Whisper‑совместимому API (можно выключить LLM, но часть функционала станет ручной).

Зависимости описаны в `requirements.txt`:

```bash
pip install -r requirements.txt
```

## Настройка окружения

Создайте файл `.env` в корне проекта (`c:\finbot\.env`) со значениями, соответствующими вашему окружению:

```env
# Telegram
TELEGRAM_BOT_TOKEN=123456789:your-telegram-bot-token

# Часовой пояс (используется для интерпретации дат)
APP_TIMEZONE=Etc/GMT-5

# Google Sheets / OAuth
GOOGLE_OAUTH_CLIENT_PATH=c:/finbot/credentials.json
GOOGLE_SHEETS_SPREADSHEET_ID=ваш_spreadsheet_id
GOOGLE_SHEETS_JOURNAL_SHEET_NAME=Журнал

# LLM (OpenAI-совместимый провайдер)
LLM_BASE_URL=https://your-llm-provider/v1
LLM_API_KEY=your-api-key
LLM_MODEL=gpt-4.1-mini
LLM_ENABLED=1  # поставьте 0, чтобы отключить LLM и работать только с ручным подтверждением

# Whisper‑модель (если провайдер поддерживает)
WHISPER_MODEL=whisper-1
```

> Примечание: путь `GOOGLE_OAUTH_CLIENT_PATH` должен указывать на JSON‑файл учётных данных OAuth клиента Google, с правами доступа к Sheets API.

## Запуск бота локально

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Проверьте, что `.env` заполнен корректно и OAuth‑файл лежит по указанному пути.

3. Запустите приложение:

   ```bash
   python -m app.main
   ```

4. Откройте Telegram и найдите своего бота по имени, которое вы задали через `@BotFather`, начните чат командой:

   ```text
   /start
   ```

5. Попробуйте отправить боту сообщение:

   ```text
   продукты 3000 вчера
   ```

   Если LLM включён и настроен, бот разберёт сумму/дату/категорию и запишет операцию в Google Sheets.  
   Если LLM отключён или не уверен в категории, бот попросит вас выбрать категорию вручную.

## Как это работает на уровне потока

1. Пользователь отправляет текст или голосовое сообщение боту в Telegram.
2. Бот (через `aiogram`) получает апдейт и передаёт его в соответствующий хэндлер.
3. Для голосового сообщения:
   - файл скачивается;
   - передаётся в Whisper‑совместимый API;
   - текст из ответа обрабатывается так же, как обычное текстовое сообщение.
4. Текст сообщения разбирается сервисом:
   - если LLM включён — вызывается GPT‑подобная модель с промптом, на выходе получаем структуру операции;
   - если LLM выключен или не справился — используется простой парсер суммы, а категория/некоторые поля остаются “pending”.
5. Собранная `Operation` сохраняется в лист “Журнал” через `JournalRepo`.
6. Если операция помечена как `pending`, бот предлагает пользователю дополнительно выбрать или скорректировать категорию через кнопки.
